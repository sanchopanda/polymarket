## Математика роста

### EV на $1 ставки

```
payout_shares = 1 / 0.65                         = 1.5385
payout_win    = shares * (1 - fee) = shares*0.98 = $1.5077
win_pnl       = +$0.5077
loss_pnl      = -$1.0
EV @ WR=70%   = 0.70 * 0.5077 + 0.30 * (-1)      = +$0.0554 / $1 (+5.54%)
EV @ WR=69% entry=0.66 = +$0.0245 / $1 (+2.45%)
```

Break-even WR: `entry / (1 - fee)` → 0.65 ⇒ 66.3%, 0.68 ⇒ 69.4%.

### Детерминистский рост (старт $20, 252 fills/день, stake=balance/30 ∈ [$1, $50])

**WR=70%, entry=0.65 (EV +5.54%):**

день 10  $2,105

**WR=69%, entry=0.66 (EV +2.45%):**
```

день 21  $1,624   ← cap достигается через 3 недели
день 30  $4,408   (~$309/день)
```

Разница параметров 1-2 п.п. → рост в 4× раз на месячном горизонте. Держать entry низким и WR высоким критично.

## Идеи/кандидаты (не реализовано)

## Итерация 2026-04-19 (per-symbol, hold-test, probe-offsets)

### Что сделали в коде

1. **`real_confirm_min_price_by_symbol: {BNB: 0.65}`** — у BNB в серой зоне 0.63-0.65 WR=44% (см. bucket-анализ ниже), поднятие порога до 0.65 срезает мусор. DOGE в этой же зоне имеет WR=69% — **не включаем** для него.
2. **`real_hold_test_by_symbol.XRP = {window_ms: 2000, sample_interval_ms: 200, min_price: 0.65}`** — оконный фильтр вместо точечной проверки. Для XRP в окне [0..2s]: bucket `min_ask ∈ [0.60, 0.63]` имеет WR=30% (n=10, −$5.19). Точечная проверка на 1s пропускает дипы на 1.5-1.7s — hold-test ловит.
   - В `engine._place_orders`: если `hold_cfg` есть — цикл `sleep(interval)` × N, собираем samples, `min(samples) ≥ min_price`. `ran_away` проверяется по last sample. Приоритет hold-test над `real_confirm_min_price` для того же символа.
3. **Probe offsets**: было 26 × 0.2s (0.2..5.0s), стало 8 точек [0.2, 0.5, 0.7, 1.0, 1.2, 1.5, 1.7, 2.0]s — плотная сетка вокруг момента принятия решения.

### Известные ограничения анализа

- Grid search на 13ч окне игнорировал `skip_miss_above_0.68`: при delay=2s ~15-30% сделок имеют ask уже выше нашего лимита 0.68 → fill не состоится, но в симуляции считалось как win/lose по исходу рынка. Честная модель резко снижает edge 2s-конфигов.
