## Сравнительный анализ площадок

| Метрика | Polymarket | Kalshi |
|---------|-----------|--------|
| Slippage от стакана | 0.02 (2¢) | 0 |
| Slippage от WS/API цены | 0.03 (3¢) | 0.02 (2¢) |
| Скорость ордера | 650-930ms | 514ms |
| Резолюция после экспирации | 146-235с | 0.9с |
| Redeem | ручной on-chain (3-5с + gas) | автоматический |
| Комиссия при p=0.65 | ~1.2% | ~2.6% |
| Формула комиссии | shares × p × 0.25 × (p(1-p))² | ceil(7% × contracts × p × (1-p)) |

## Выводы для арбитражного бота

1. **Fill = book best ask** при малых ставках. Slippage от стакана практически нулевой. Решение о входе нужно принимать по стакану, а не по snapshot/WS ценам.

2. **Комиссии Kalshi ~2× выше Polymarket** на всех ценовых уровнях. Суммарная комиссия двух ног при $50 и p≈0.70: ~$1.59. Это минимальное арбитражное окно для безубыточности.

3. **Формулы комиссий в симуляции корректны** — реальные значения совпали с расчётными.

4. **Резолюция Kalshi мгновенная** (0.9с) vs Polymarket (2-4 мин). Это не влияет на арбитраж напрямую, но важно для скорости оборота капитала.

5. **Polymarket требует ручной redeem** (on-chain транзакция, gas ~0.016 POL). Kalshi settlement автоматический.

6. **Gamma API snapshot цены бесполезны** для принятия решений — отклонение до 19¢. WS цены точнее (отклонение 2-3¢), но надёжнее всего — стакан.

7. **Скорость ордеров сопоставима** (~500ms обе площадки). Достаточно быстро для 15-минутных рынков.
