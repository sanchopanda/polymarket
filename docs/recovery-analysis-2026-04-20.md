## Внедрено 2026-04-20

1. **Price-floor** `real_order_cancel_if_ask_below: 0.60` — при падении ask ниже 0.60 у working order отменяем (ловит кейсы типа SOL fill@0.52).
2. **Time-to-expiry filter** `max_seconds_to_expiry: 240` в обоих `five_minute`/`fifteen_minute`. Анализ показал: XRP bucket [240,300]с WR=29%, SOL [180,240] WR=58%. Баланс: cut 49 trades WR=63% −$2.06, kept 252 WR=69% +$7.94.
3. **Per-symbol delay** `real_confirm_delay_seconds_by_symbol: {SOL: 2.0, DOGE: 2.0, BNB: 2.0}` — альтам 2s вместо 1s. BTC/ETH остаются 1s.
4. **DOGE min override** добавлен: `real_confirm_min_price_by_symbol: {DOGE: 0.65, BNB: 0.65}`.

## Анализ ограничений

- Симуляция строится на `price_probes` (kind='signal') — по 8 offsets [0.2, 0.5, 0.7, 1.0, 1.2, 1.5, 1.7, 2.0]s. Для точки 2000ms берётся ближайший offset. Небольшая погрешность (~ $2).
- Slow-fill положения (ask@2s > 0.68) в реальности всё равно заполняются когда ask возвращается. Сим их тоже включает через actual PnL — корректно.
- Hypothetical PnL для skipped_filter считается fill at top_price=0.68 — оптимистично. Использовался только для обоснования направлений, не для абсолютных цифр.
